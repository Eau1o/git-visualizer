# Git 可视化教室 — 设计文档

## 项目概述

一个交互式 Git 可视化教学网页，通过 Canvas 动画直观展示 Git 各项功能的工作流程，帮助初学者理解 Git 的核心概念和操作。

### 核心理念
- **所见即所得**：每个 Git 操作都在 Canvas 上实时动画展示
- **故事驱动**：通过具体的开发场景来学习，而不是枯燥的命令列表
- **循序渐进**：MVP → 全面覆盖 → 完整百科，迭代开发

---

## MVP 范围

四组基础 Git 功能：

| 场景组 | 功能 | 学习目标 |
|--------|------|----------|
| ① 本地仓库基础 | init, add, commit, status, log | 理解工作区→暂存区→仓库的流转 |
| ② 分支操作 | branch, checkout/switch, merge | 理解分支分叉与合并 |
| ③ 远程协作 | remote, push, pull, clone | 理解本地与远程的同步 |
| ④ 撤销/修复 | restore, reset, revert | 理解如何安全地撤销操作 |

---

## 技术方案

- **技术栈**：纯 HTML + CSS + JavaScript + Canvas API
- **架构**：单页应用 (SPA)，无外部依赖
- **渲染**：Canvas 2D API 绘制 Git 仓库状态图，CSS 过渡配合实现 UI 动画

### 项目文件结构 (MVP)

```
git-visualizer/
├── index.html              # 主页面
├── css/
│   └── style.css           # 全局样式（Git 橙主题）
├── js/
│   ├── app.js              # 应用入口，布局初始化

```

## 页面布局

采用三栏式混合布局：

```
┌──────────────────────────────────────────────────────────┐
│  🧭 顶部导航栏：标题 + 进度指示器 (2/12) + Git 橙主题     │
├────────────┬─────────────────────────┬────────────────────┤
│  📋 左侧    │   🎨 中间 (Canvas)      │   🔧 右侧          │
│  场景引导面板 │   Git 仓库可视化区域     │   命令面板          │
│            │                         │                    │
│  ✅ 步骤 1   │   ○ → ○ → ○           │   git init  ✅     │
│  ▶ 步骤 2   │   ↙      ↘            │   git add          │
│  步骤 3     │  ○ ← → ○              │   git commit       │
│  步骤 4     │                        │   git branch       │
│            │                         │                    │
├────────────┴─────────────────────────┴────────────────────┤
│  💬 底部输出面板：命令执行结果/日志                        │
└──────────────────────────────────────────────────────────┘
```

### 各区域说明

**左侧 — 场景引导面板**
- 当前场景的步骤列表
- 已完成步骤显示 ✅，当前步骤高亮 ▶
- 点击可跳转到已完成步骤回顾
- 顶部有场景切换（基础/分支/远程/撤销）

**中间 — Canvas 可视化区域 (核心)**
- 深色背景 (#1a1a2e)，呈现 Git 仓库状态
- 三区状态栏：工作区 | 暂存区 | 本地仓库
- Commit 节点：圆形 + SHA 缩略 + 提交信息
- 分支标签：圆角 badge，不同分支不同颜色
- 连线：commit 历史线，合并时显示分支分叉
- HEAD 指针：显示当前所在位置

**右侧 — 命令面板**
- 当前可用命令按钮，按场景分组
- 可用状态：橙色高亮，带悬停效果
- 不可用状态：灰色禁点
- 点击后触发对应 Git 操作的动画

**底部 — 输出日志**
- 终端风格的命令回显
- 显示命令执行结果（成功/失败信息）
- 自动滚动到最新输出

## Canvas 可视化元素设计

### 基础图元

1. **Commit 节点**
   - 圆形，半径 ~20px
   - 渐变填充，带阴影
   - 点击显示详情：SHA、作者、时间、提交信息
   - 新节点出现时带脉冲放大动画

2. **分支标签**
   - 圆角矩形 badge
   - main 分支：橙色 (#f05032)
   - 功能分支：绿色 (#27ae60)
   - 其他分支：按需分配颜色
   - 指向对应的 commit 节点

3. **提交历史线**
   - 连接 commit 节点的曲线/折线
   - 主线为实线，分支线使用不同颜色
   - 合并时两条线汇入一个节点

4. **三区状态**
   - 工作区：显示未跟踪/已修改文件列表（红色标签）
   - 暂存区：显示已暂存文件列表（黄色标签）
   - 仓库：显示 commit 节点链

5. **远程仓库**
   - 云端图标 (☁️)
   - 与本地仓库之间有双向箭头
   - push/pull 时显示节点在两者间移动

### 核心动画

| 操作 | 动画效果 | 时长 |
|------|----------|------|
| git init | 空白画布上出现第一个空仓库节点 | 0.8s |
| git add | 文件标签从工作区飞入暂存区，颜色从红变黄 | 0.6s |
| git commit | 新节点从时间线末端弹出，脉冲光效，分支标签更新 | 1.0s |
| git branch | 新分支标签在当前位置出现 | 0.5s |
| git checkout | 分支标签高亮，HEAD 指针移动到目标分支 | 0.5s |
| git merge | 两条分支线延伸汇合，生成合并节点，闪烁提示 | 1.2s |
| git push | 本地节点序列飞向远程仓库图标 | 1.0s |
| git pull | 远程节点飞入本地，合并到当前分支 | 1.0s |
| git restore | 文件从工作区消失（撤销修改） | 0.5s |
| git reset | commit 节点淡出/分支标签回退 | 0.8s |
| git revert | 新增一个"撤销提交"节点 | 0.8s |

## 场景与步骤设计 (MVP)

### 场景 1：本地仓库基础 — "我的第一个项目"

| 步骤 | 操作 | 可视化效果 | 教学要点 |
|------|------|-----------|----------|
| 1.1 | 创建文件 | 工作区出现未跟踪文件(红色) | 认识工作区 |
| 1.2 | git init | 出现空的仓库节点 | 初始化仓库 |
| 1.3 | git status | 输出显示未跟踪文件 | 查看状态 |
| 1.4 | git add | 文件从工作区飞入暂存区(变绿) | 暂存的概念 |
| 1.5 | git commit | 生成第一个 commit 节点 | 提交的概念 |
| 1.6 | git log | 显示 commit 历史链 | 查看历史 |

### 场景 2：分支操作 — "同时开发两个功能"

| 步骤 | 操作 | 可视化效果 | 教学要点 |
|------|------|-----------|----------|
| 2.1 | 在 main 上做两次提交 | 两个 commit 节点串联 | 准备分支场景 |
| 2.2 | git branch feature-a | 绿色分支标签出现在最新节点 | 创建分支 |
| 2.3 | git checkout feature-a | HEAD 移到 feature-a | 切换分支 |
| 2.4 | 在 feature-a 上提交 | 分支线分叉，feature-a 前进 | 分支分叉 |
| 2.5 | git checkout main | HEAD 移回 main | 回到主分支 |
| 2.6 | git merge feature-a | 两条线汇合成合并节点 | 合并概念 |

### 场景 3：远程协作 — "和团队一起工作"

| 步骤 | 操作 | 可视化效果 | 教学要点 |
|------|------|-----------|----------|
| 3.1 | git remote add origin | 出现远程仓库云图标 | 远程仓库概念 |
| 3.2 | git push | 本地节点序列飞向云端 | 推送代码 |
| 3.3 | (模拟队友提交) | 远程仓库出现新节点 | 远程进展 |
| 3.4 | git pull | 远程节点飞入并合并 | 拉取更新 |
| 3.5 | git clone | 完整复制一份仓库 | 克隆仓库 |

### 场景 4：撤销/修复 — "犯错怎么办"

| 步骤 | 操作 | 可视化效果 | 教学要点 |
|------|------|-----------|----------|
| 4.1 | 修改已提交文件 | 文件变红色(已修改) | 了解修改状态 |
| 4.2 | git restore | 文件恢复原样(修改消失) | 撤销工作区修改 |
| 4.3 | git add → 发现错误 | 文件在暂存区 | 错误暂存 |
| 4.4 | git reset | 文件回到工作区 | 取消暂存 |
| 4.5 | git commit → 发现错误 | 错误的 commit 节点 | 错误提交 |
| 4.6 | git revert | 生成新的撤销节点，原节点保留 | 安全撤销 |

## 交互设计

### 用户操作流程
1. 用户从左侧选择场景（默认从场景 1 开始）
2. 场景加载后，右侧显示当前步骤可用的命令
3. 用户点击命令按钮 → 触发动效 → Canvas 更新 → 输出日志
4. 完成后当前步骤标记 ✅，下一步自动解锁
5. 用户可随时回到已完成步骤回顾

### 自由探索模式
- 所有已解锁的命令按钮在右侧可见
- 用户可以跨步骤点击已学过的命令
- 系统会阻止产生矛盾的操作（如未 init 就 commit）

### 引导提示
- 当前步骤有文字提示（"试试点击 git add"）
- 可用命令按钮有呼吸动画吸引注意
- 关键步骤有解释弹窗/气泡提示

- 所有代码为纯前端，无后端依赖

- 响应式设计，适配桌面端（主要为桌面使用）
